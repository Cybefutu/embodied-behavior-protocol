# Specification

The current normative core lives in [v0.1/core.md](v0.1/core.md).

The broader design proposal lives in [v0.1/SPEC.md](v0.1/SPEC.md).

As OEBP matures, the large proposal should be split into smaller normative
documents for the core model, behavior contracts, composition, capability
matching, lifecycle, trace and dataset annotation, safety, extension rules, and
bindings.

Normative changes should go through the RFC process in `governance/`.
