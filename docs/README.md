# Documentation

This directory contains non-normative supporting material for OEBP.

- `research-basis.md` summarizes related work and interoperability targets.
- `glossary.md` defines recurring terms used by the specification and roadmap.
- `ARCHITECTURE_REVIEW.md` records design rationale and the v0.1 scope boundary.
- `bindings/` documents non-normative JSON, Protobuf, ROS 2, and behavior-tree
  binding guidance.
- `training-data.md` explains how to create training-data views from OEBP
  episode annotations without embedding raw media or action streams.

Normative requirements belong under `spec/` and should be linked from the
relevant RFCs or ADRs.
