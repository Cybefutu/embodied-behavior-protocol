# Conformance Fixtures

This directory contains schema fixtures for OEBP v0.1.

The fixture manifest records each document, the schema it targets, whether it is
expected to validate, and the reason for that expectation. Invalid fixtures also
declare `expected_error_code` so future validators can map schema failures to
stable machine-readable findings.

M1-O3 requires at least 20 valid fixtures and 50 invalid fixtures. The current
corpus is generated by:

```bash
python3 scripts/generate_core_fixtures.py
```

Generated fixtures cover the protocol envelope, context snapshot, predicate
expression, skill contract, behavior spec, capability profile, invocation
request, invocation feedback, invocation result, trace span, episode annotation,
and provenance record schemas.

Run:

```bash
python3 scripts/check_fixtures.py
```
